01

Vectors

A vector is an ordered list of numbers. In ML, vectors represent things — a word, a data point, an image patch, a user's preferences.

Python · NumPy
import numpy as np

# A user's movie ratings (action, romance, thriller)
user  = np.array([5, 1, 4])
user2 = np.array([4, 1, 5])

# Magnitude (length) of a vector
magnitude = np.linalg.norm(user)   # sqrt(5² + 1² + 4²) = 6.48

Geometrically, a vector is an arrow in space. Its direction encodes the "type" of thing it represents; its magnitude encodes the "strength."

02

Matrices

A matrix is a 2D grid of numbers — a stack of vectors. In ML: weight layers in neural nets, batches of training examples, attention scores in transformers.

Python · NumPy
# 3 users, each with 3 ratings
ratings = np.array([
    [5, 1, 4],   # user 0
    [4, 1, 5],   # user 1
    [1, 5, 2],   # user 2
])

print(ratings.shape)    # (3, 3)  → 3 rows, 3 cols
print(ratings[1])       # [4, 1, 5]  → user 1's ratings
print(ratings[:, 0])   # [5, 4, 1]  → all users' action ratings

# Matrix multiplication — the core of every neural net layer
W = np.array([[1, 0], [0, 2], [1, 1]])   # shape (3, 2)
x = np.array([3, 4])                      # shape (2,)
out = W @ x   # shape (3,) → [3, 8, 7]
Key insight

Think of W @ x as: "transform the input x into a new space." This is exactly what every neural network layer does.

03

Dot Product

The dot product multiplies element-wise then sums. It is the most important operation in ML — it measures similarity.

a · b = a₁b₁ + a₂b₂ + a₃b₃ = ||a|| ||b|| cos(θ)
Python · NumPy
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

dot = a @ b   # 1*4 + 2*5 + 3*6 = 32

# Cosine similarity = dot product of unit vectors
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

user0 = np.array([5, 1, 4])
user1 = np.array([4, 1, 5])
user2 = np.array([1, 5, 2])

print(cosine_similarity(user0, user1))  # 0.97 — very similar!
print(cosine_similarity(user0, user2))  # 0.29 — very different

🔗 Connection to LLMs

The dot product is the core of attention in transformers (query · key = how much to attend), RAG retrieval (find most similar embedding), and recommendation systems (find similar users). This one operation underlies all of it.

04

Gradient Descent

Gradient descent is how every ML model learns. You measure how wrong the model is (loss function), then nudge the weights in the direction that reduces the loss (negative gradient). Repeat thousands of times.

The core loop

Python · NumPy — Linear Regression from scratch
np.random.seed(42)
X = np.linspace(0, 10, 100)
y = 2 * X + 1 + np.random.randn(100) * 2

w  = 0.0    # slope   (want to learn w ≈ 2)
b  = 0.0    # intercept (want to learn b ≈ 1)
lr = 0.001  # learning rate

for epoch in range(300):
    # 1. FORWARD PASS
    y_pred = w * X + b

    # 2. LOSS (Mean Squared Error)
    loss = np.mean((y_pred - y) ** 2)

    # 3. GRADIENTS (chain rule)
    dL_dw = np.mean(2 * (y_pred - y) * X)
    dL_db = np.mean(2 * (y_pred - y))

    # 4. UPDATE (step opposite to gradient)
    w = w - lr * dL_dw
    b = b - lr * dL_db

print(f"Learned: w = {w:.3f}, b = {b:.3f}")
# → Learned: w = 1.989, b = 1.282  ✅

Gradient formulas (from chain rule)

dL/dw = (2/n) × Σ (ŷᵢ − yᵢ) × xᵢ     dL/db = (2/n) × Σ (ŷᵢ − yᵢ)

The × xᵢ term in dL/dw means: a weight connected to a large input gets a larger gradient update. This is why input normalization matters.

ConceptWhat it really means
Loss functionOne number that says "how wrong are you right now"
GradientThe slope — which direction increases the loss
Learning rateStep size. Too big = overshoots. Too small = takes forever
EpochOne full pass over all training data

Probability Basics

Probability is the language models speak. Every LLM output is a probability distribution over tokens.

Distributions you must know

Python · NumPy
# 1. Normal (Gaussian) — used for weight initialization, noise
samples = np.random.normal(loc=0, scale=1, size=10000)
# 68% within 1σ, 95% within 2σ, 99.7% within 3σ

# 2. Bernoulli — binary outcome (spam/not-spam)
flips = np.random.binomial(n=1, p=0.7, size=1000)

# 3. Uniform — used in weight initialization
weights = np.random.uniform(low=-0.1, high=0.1, size=(3, 3))

Bayes' Theorem

P(A|B) = P(B|A) × P(A) / P(B)
Python — Medical test example
P_disease             = 0.01   # 1% of population has it
P_pos_given_disease   = 0.95   # test catches 95% of cases
P_pos_given_no_disease = 0.05  # 5% false positive rate

P_positive = (P_pos_given_disease * P_disease +
              P_pos_given_no_disease * (1 - P_disease))

P_disease_given_pos = (P_pos_given_disease * P_disease) / P_positive

print(f"P(disease | positive test) = {P_disease_given_pos:.3f}")
# → 0.161  ← only 16%! Base rate neglect.
Why this matters for LLMs

A model's stated confidence must be weighted against how common the correct answer actually is. A model that says "I'm 99% sure" and is wrong is catastrophically worse than one that says "I'm 50% sure."

06

The Sigmoid Function

Sigmoid squashes any real number into the range (0, 1), turning raw model scores into probabilities.

σ(z) = 1 / (1 + e⁻ᶻ)
Python
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

print(sigmoid(-10))   # ≈ 0.00005  — confident: class 0
print(sigmoid(0))     # 0.5       — completely uncertain
print(sigmoid(10))    # ≈ 0.99995 — confident: class 1

# Softmax — generalized sigmoid for K classes (used in LLMs)
def softmax(z):
    e = np.exp(z - z.max())   # subtract max for stability
    return e / e.sum()

logits = np.array([2.1, 0.5, -1.2, 3.0, 1.1])
probs  = softmax(logits)  # sums to 1.0

# Temperature divides logits before softmax
print(softmax(logits / 0.3).round(3))  # sharper (low temp)
print(softmax(logits / 2.0).round(3))  # flatter (high temp)

🔗 Connection to LLMs — Temperature

The LLM final layer is softmax(W @ hidden_state + b) over 100,000+ tokens. Temperature divides the logits before softmax — low temperature makes the distribution sharper (more deterministic), high temperature makes it flatter (more random). This is the same math you just learned.

Cross-entropy loss

MSE fails for classification (non-convex surface). Cross-entropy penalizes confident wrong answers exponentially harder.

Python
def cross_entropy(y_true, y_pred):
    y_pred = np.clip(y_pred, 1e-9, 1 - 1e-9)
    return -np.mean(
        y_true * np.log(y_pred) +
        (1 - y_true) * np.log(1 - y_pred)
    )

print(cross_entropy(1, 0.99))  # 0.01  — correct, confident
print(cross_entropy(1, 0.50))  # 0.69  — correct, uncertain
print(cross_entropy(1, 0.01))  # 4.60  — WRONG, confident ← huge!

Evaluation metrics

MetricFormulaUse when
PrecisionTP / (TP + FP)Cost of false positives is high (spam filter)
RecallTP / (TP + FN)Cost of false negatives is high (cancer detection)
F12 × (P × R) / (P + R)Imbalanced classes, want balance of both
AUC-ROCArea under ROC curveThreshold-independent ranking quality
Python — Full logistic regression from scratch
w = np.zeros(X_train.shape[1])
b = 0.0; lr = 0.1

for epoch in range(300):
    z      = X_train @ w + b
    y_pred = sigmoid(z)
    error  = y_pred - y_train
    dw     = X_train.T @ error / len(y_train)
    db     = error.mean()
    w -= lr * dw
    b -= lr * db

# Evaluate
y_test_pred  = sigmoid(X_test @ w + b)
y_test_class = (y_test_pred >= 0.5).astype(int)
accuracy     = (y_test_class == y_test).mean()
print(f"Test accuracy: {accuracy:.3f}")   # → ~0.90+
08

Decision Trees

A decision tree learns a series of yes/no questions that split data into pure groups. Completely interpretable — you can read the exact rules it learned.

Python · sklearn
from sklearn.tree import DecisionTreeClassifier, export_text

tree = DecisionTreeClassifier(max_depth=3)
tree.fit(X, y)
print(export_text(tree, feature_names=['feat_0', 'feat_1']))
# |--- feat_1 <= 0.23
# |   |--- feat_0 <= -1.54  → class: 0
# |   |--- feat_0 >  -1.54  → class: 1
# |--- feat_1 >  0.23       → class: 1

Impurity measures

Python
def gini(p):
    """0 = pure node, 0.5 = maximally mixed"""
    return 1 - (p**2 + (1-p)**2)

def entropy(p):
    """0 = pure, 1 = maximally mixed"""
    if p == 0 or p == 1: return 0
    return -(p * np.log2(p) + (1-p) * np.log2(1-p))

print(gini(1.0))   # 0.0 ← perfectly pure
print(gini(0.5))   # 0.5 ← maximally mixed

At each node, the tree tries every feature and every threshold, picking the split that reduces impurity the most (highest information gain).

09

Random Forests

A single deep tree memorizes training data. The fix: train hundreds of slightly different trees and average their predictions. Errors cancel out.

Python · sklearn
from sklearn.ensemble import RandomForestClassifier

# Single deep tree — overfits
tree = DecisionTreeClassifier(max_depth=None)
tree.fit(X_train, y_train)
print(tree.score(X_train, y_train), tree.score(X_test, y_test))
# → 1.000, 0.817  ← memorized training data!

# Random forest — generalizes
rf = RandomForestClassifier(
    n_estimators=100,
    bootstrap=True,      # each tree sees ~63% of data
    max_features='sqrt', # only sqrt(n_features) at each split
    random_state=42
)
rf.fit(X_train, y_train)
print(rf.score(X_train, y_train), rf.score(X_test, y_test))
# → 0.979, 0.917  ← much better generalization

# Feature importance
for i, imp in enumerate(rf.feature_importances_):
    print(f"feature_{i}: {imp:.3f}")
Why forests work

Bootstrap sampling + feature subsampling = decorrelated trees. Each tree makes different errors. Averaging cancels the errors. This is ensemble learning — the same principle behind Mixture-of-Experts models like Mixtral and GPT-4.

10

Overfitting

The central problem of ML. A model that memorizes training data — learns the noise, not the signal — will fail on new data. Everything in ML exists to fight this.

Python — Polynomial overfitting
from sklearn.preprocessing import PolynomialFeatures
from sklearn.pipeline import make_pipeline

for degree in [1, 4, 15]:
    model = make_pipeline(PolynomialFeatures(degree), LinearRegression())
    model.fit(X_raw, y_raw)
    train_err = np.mean((model.predict(X_raw) - y_raw)**2)
    test_err  = np.mean((model.predict(X_test) - y_true)**2)
    print(f"degree={degree:2d}  train={train_err:.4f}  test={test_err:.4f}")

# degree= 1  train=0.2028  test=0.2217  ← underfit
# degree= 4  train=0.0679  test=0.0702  ← just right ✓
# degree=15  train=0.0041  test=1.9873  ← OVERFIT

Train / Validation / Test splits

SplitSizePurpose
Training set~70%Model learns from this
Validation set~15%Tune hyperparameters, catch overfitting
Test set~15%Final evaluation — touch exactly once
Data leakage — the silent killer

Touching the test set while tuning leaks information — your model overfits to the test set itself. The validation set is your proxy for the real world during development. Test set is sacred.

Regularization

Regularization adds a penalty for complexity to the loss function. The model is forced to stay simple unless the data strongly justifies complexity.

Python · sklearn
from sklearn.linear_model import Ridge, Lasso

# L2 (Ridge): Loss = MSE + α × Σ(wᵢ²)
# Pushes all weights toward zero, none to exactly zero
ridge = Ridge(alpha=1.0)

# L1 (Lasso): Loss = MSE + α × Σ|wᵢ|
# Pushes some weights to EXACTLY zero → feature selection
lasso = Lasso(alpha=0.1)

# Early stopping — simplest regularization
model_es = MLPClassifier(
    early_stopping=True,
    n_iter_no_change=10,   # stop if no improvement for 10 epochs
)
L1 (Lasso)L2 (Ridge)
PenaltyΣ|w|Σw²
EffectSparse weights (some = 0)Small weights (none = 0)
Use whenMany irrelevant featuresAll features matter a bit
LLM equivalentPruningWeight decay (AdamW)

🔗 Connection to LLMs

L2 regularization (weight decay) is used in every LLM optimizer. AdamW = Adam + L2 weight decay. Every major model (GPT, Claude, Llama) trains with weight decay. The exact same math you see in Ridge regression.

12

Cross-Validation & Hyperparameter Tuning

A single train/test split can be lucky or unlucky. Cross-validation rotates the validation set, giving a stable estimate of true performance.

Python · sklearn
from sklearn.model_selection import cross_val_score, GridSearchCV

# 5-fold CV: split into 5 chunks, each gets to be val set once
cv_scores = cross_val_score(rf, X, y, cv=5, scoring='f1')
print(f"Mean F1: {cv_scores.mean():.3f} ± {cv_scores.std():.3f}")
# Low std = stable model, not just lucky

# GridSearchCV — try every hyperparameter combination
param_grid = {
    'n_estimators':  [50, 100, 200],
    'max_depth':     [3, 5, 10, None],
    'max_features':  ['sqrt', 'log2'],
}
gs = GridSearchCV(rf, param_grid, cv=5, scoring='f1', n_jobs=-1)
gs.fit(X_train, y_train)
print(gs.best_params_, gs.best_score_)
13

The Full ML Pipeline

This is the pattern for the Week 2 Kaggle project — and for every serious ML project you'll ever build.

Python · sklearn — Production pattern
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

# Step 1: Build the pipeline
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('clf',    RandomForestClassifier(random_state=42)),
])

# Step 2: Define hyperparameters (prefix with step name + __)
params = {
    'clf__n_estimators': [100, 200],
    'clf__max_depth':    [5, 10, None],
}

# Step 3: Search with cross-validation
gs = GridSearchCV(pipe, params, cv=5, scoring='f1', n_jobs=-1)
gs.fit(X_train, y_train)

# Step 4: Evaluate on test set — ONCE, at the very end
print(f"Best params:  {gs.best_params_}")
print(f"CV F1:        {gs.best_score_:.3f}")
print(f"Test F1:      {gs.score(X_test, y_test):.3f}")
