Here's an honest answer — yes, there's a lot left that the 20-week roadmap intentionally doesn't cover, because it's scoped to get you hired as an AI engineer, not to make you an expert in everything.

What the roadmap covers well ✅

The 20 weeks gives you a solid, hireable foundation in:

  • ML fundamentals, transformers, and how LLMs work
  • Prompt engineering with scientific measurement
  • RAG systems end-to-end with evaluation
  • Agent architectures (ReAct, memory, LangGraph, multi-agent, MCP)
  • Production deployment basics

What's missing — by category

🔬 Model Training & Fine-tuning

The roadmap only uses pre-trained models via API. You haven't touched:

  • Fine-tuning — LoRA, QLoRA, full fine-tuning on custom datasets
  • SFT / RLHF / DPO — how instruction-following is actually trained into models
  • Hugging Face Transformers — loading, training, pushing your own models
  • Model evaluation — MMLU, HeliaSwag, custom benchmarks
  • Inference optimization — quantization (GGUF, AWQ), vLLM, TensorRT

This is a big gap if your target company trains or fine-tunes models.

🏗️ Advanced RAG

The roadmap covers the core pipeline. Missing:

  • GraphRAG — Microsoft's graph-based retrieval, better for multi-hop reasoning
  • HyDE (Hypothetical Document Embeddings) — generate fake answer first, then search
  • Multi-vector retrieval — parent-child chunking, summary + full doc indexing
  • Agentic RAG — agent decides when/what to retrieve, not a fixed pipeline
  • Colpali — vision-based RAG for PDFs with figures and tables

🤖 Advanced Agent Topics

  • Long-horizon planning — agents that plan over 50+ steps reliably
  • Mixture of Agents — routing between specialized models
  • OpenAI Swarm / AutoGen v2 — alternative multi-agent frameworks
  • Agent benchmarks — SWE-bench, WebArena, GAIA — how to measure agent capability properly
  • Function calling vs tool use — OpenAI's model vs Anthropic's, tradeoffs

☁️ MLOps & Infrastructure

  • Model serving — vLLM, Triton, BentoML, Modal for GPU inference
  • Vector DB at scale — Pinecone, Weaviate, Qdrant internals, indexing strategies at 100M+ docs
  • Feature stores — connecting ML pipelines to production data
  • Experiment tracking — MLflow, Weights & Biases beyond basic logging
  • CI/CD for ML — DVC, data versioning, model registry workflows

🔐 AI Safety & Alignment (growing area)

  • Red-teaming LLMs — systematic adversarial testing
  • Constitutional AI — how Claude is trained to be helpful/harmless
  • Interpretability basics — attention visualization, probing classifiers
  • Bias evaluation — measuring and mitigating demographic bias in model outputs

📊 Data Engineering for AI

  • Dataset curation — how to build high-quality training datasets
  • Synthetic data generation — using LLMs to generate training data
  • Data flywheel — using production traffic to improve your model
  • Annotation workflows — Label Studio, human-in-the-loop pipelines at scale

🎨 Multimodal

  • Vision-language models — Claude Vision, GPT-4V, LLaVA
  • Image/video generation — Stable Diffusion, ComfyUI pipelines
  • Audio models — Whisper (transcription), TTS, voice agents
  • Multimodal RAG — retrieving over images, tables, charts

💡 Emerging (2025 frontier)

  • Reasoning models — o1/o3-style extended thinking, how to use them differently
  • Computer use / browser agents — Claude Computer Use, real-world automation
  • Long context techniques — effective use of 1M+ context windows, needle-in-a-haystack eval
  • Structured generation — constrained decoding with Outlines/Guidance
  • Memory systems at scale — Mem0, MemoryOS, cognitive architectures

Honest prioritization

If you're interviewing for AI Engineer / LLM Engineer roles at most companies right now, the 20-week roadmap covers ~80% of what gets tested. The gaps that matter most to close first, in order:

  1. Fine-tuning with LoRA/QLoRA — comes up in ~40% of AI engineer interviews
  2. vLLM or model serving — if you want infra-leaning roles
  3. Advanced RAG patterns (HyDE, parent-child chunking) — quick wins on top of what you know
  4. Reasoning models (o1/o3 usage patterns) — emerging but already in interviews

Everything else is specialization — pick based on where you want to go (research, infra, application, safety).

The roadmap is genuinely complete for getting hired. These are the things that take you from hired to excellent over the next year on the job.
